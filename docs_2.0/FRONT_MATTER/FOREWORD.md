# Foreword

This book intentionally does not include a traditional foreword.

A real foreword should be written by someone other than the author.
It should add outside perspective, not repeat internal project beliefs.

At the moment, forcing a foreword would create one of two bad outcomes:

- a fake external voice
- an internal promotional note pretending to be independent

Neither is useful.

So the honest position is simple:

- no outside foreword has been commissioned yet
- no surrogate foreword is being fabricated for polish

If a future edition includes a foreword, it should come from someone who can speak credibly about:

- the need Stroid is trying to address
- the tradeoffs of its architecture
- where it fits relative to other state-management tools

Until that exists, this page stays deliberately minimal.
