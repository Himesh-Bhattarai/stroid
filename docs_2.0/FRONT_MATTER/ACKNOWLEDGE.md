# Acknowledgments

This project was not built in isolation, even if much of the work was done by one person.

The code, testing, structure, and writing all benefited from a mix of:

- repeated experimentation
- open-source ecosystem learning
- existing state-management libraries that set a high bar
- AI-assisted exploration and review

## Honest Acknowledgment of AI Use

AI was used heavily in this project.

That should not be hidden.

It helped with:

- implementation directions
- code review pressure
- edge-case analysis
- documentation writing
- benchmark framing
- architecture questioning

But AI did not replace judgment.

The important work remained human:

- deciding what behavior should exist
- deciding which tradeoffs were acceptable
- deciding which fixes were real and which were noise
- deciding what belonged in the public API

## Technical Debt Awareness

Acknowledgment also means admitting that the project improved through friction, not through a perfect first plan.

Many of the strongest decisions in Stroid came from:

- bugs
- failed assumptions
- refactors
- size pressure
- boundary problems
