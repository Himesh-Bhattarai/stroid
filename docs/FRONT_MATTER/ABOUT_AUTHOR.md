# About the Author

Himesh Chanchal Bhattarai is the builder behind Stroid.

This project did not come from a polished package-lab background.
It came from repeated practical frustration with state-management friction:

- repeated setup
- repeated support code
- repeated debugging of async and update behavior
- repeated architectural drag around state systems

The most important thing to know about the author in the context of this documentation is not a title.

It is the pattern of attention behind the work.

The Stroid project reflects someone who keeps pushing on questions like:

- what actually breaks under pressure?
- what hidden assumption is carrying this behavior?
- what still fails even when the happy-path demo works?
- what belongs in the package, and what should be cut?

That mindset shaped both the code and the writing.

This documentation is therefore not trying to sound like a neutral product brochure.
It is written from the position of a builder who wants the system to be legible, defensible, and honest.
