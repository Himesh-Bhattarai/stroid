# Security Policy

Thank you for helping keep **Stroid** secure.

---

## Supported Versions

Security fixes are provided for the latest `0.1.x` line.

| Version | Supported |
| ------- | --------- |
| `0.1.x` | :white_check_mark: |
| `<0.1.0` | :x: |

---

## Reporting a Vulnerability

Please **do not** report security vulnerabilities in public issues.

Use GitHub's private vulnerability reporting:

- Open a private advisory draft: <https://github.com/Himesh-Bhattarai/stroid/security/advisories/new>
- Include:
  - Affected version(s)
  - Impact and severity estimate
  - Reproduction steps or proof of concept
  - Any suggested mitigation or patch idea

If private reporting is unavailable, open an issue titled:

`[SECURITY] Private report requested`

Do not include exploit details in that public issue.

---

## Response Process

- Acknowledgement target: within 72 hours
- Initial triage target: within 7 days
- Fix and disclosure timeline: based on severity and ecosystem impact

We will coordinate disclosure and release notes once a fix is available.
